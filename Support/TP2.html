TP2 – Programmation événementielle

Introduction

L'objectif de ce TP est de manipuler les événements Javascript.

Nous programmerons une horloge munie d'alarmes configurables.

Nous commencerons avec une version simple que nous complèterons progressivement.

Exercice 1 - Horloge simple

Dans un dossier DAI/TP2/, enregistrez les fichiers clock_v1.html , clock.css et clock_v1.js .

Modifiez le fichier clock_v1.js selon les étapes suivantes.

Étape 1 - Affichage de l'heure

Afficher l'heure courante avec le format "hh:mm:ss" sur la page.

L'objet Date possède des méthodes pour récupérer les heures, minutes et secondes.

Vous veillerez à formatter les valeurs numériques sur deux chiffres.

Étape 2 - Mise à jour régulière de l'heure

Vous utiliserez la fonction window.SetInterval qui permet de lancer une fonction à intervalles réguliers.

Étape 3 - Mise en place d'une pause

Si la pause est enclenchée (propriété checked de la checkbox) alors la mise à jour régulière de l'heure est suspendue. Elle reprend quand la pause est déselectionnée.

Aperçu du résultat

Une fois terminée, votre application Web ressemblera à celle ci-dessous

Exercice 2 - Horloge avec alarmes réglables

Dans le dossier DAI/TP2/, enregistrez les fichiers clock_v2.html , clock.css (oui, le remplacer) et clock_v2.js .

Modifiez le fichier clock_v2.js selon les étapes suivantes.

Étape 1 - Affichage et pause de l'heure

Dans le fichier clock_v2.js, incorporez le code que vous avez écrit au cours de l'exercice 1 afin d'avoir l'affichage continu de l'heure et la possibilité de le mettre en pause.

Étape 2 - Remplissage des listes déroulantes

Complétez les code afin de remplir les listes déroulante des heures et minutes, par pas de 1.

Étape 3 - Mise en place d'une première alarme

Modifiez la fonction enclencherAlarme() pour qu'une alarme se déclenche à l'heure choisie quand la checkbox est selectionnée.

Une boite de dialogue apparaît alors et affiche la description correspondante.

La sélection de la checkbox grise les champs de l'alarme, qui ne sont plus modifiables. Seul le bouton "Enlever alarme" reste cliquable.

La désélection de la checkbox :

  • annule l'alarme qui avait été enclenchée ;
  • dégrise les champs de l'alarme.

Étape 4 - Ajout d'alarmes supplémentaires

Modifier le code afin que l'appui sur le bouton "Ajouter alarme" remplisse le nouveau noeud inséré avec le code HTML de la première alarme.

Étape 5 - Suppression d'alarmes

En cliquant sur un bouton "Enlever alarme", la ligne correpondante disparaît.

Veillez toutefois à ce qu'il reste toujours au moins une alarme.

Quand une alarme est supprimée, elle doit désactiver l'alarme correspondante qui avait éventuellement été enclenchée.

Aperçu du résultat
